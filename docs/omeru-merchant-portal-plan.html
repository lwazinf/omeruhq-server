03
Page Structure

13 pages. Each page has a defined purpose, layout, and visual spec. All inherit the sidebar + off-white content area layout unless noted.

Login
/login
New

Standalone auth page — no sidebar. Merchants enter their WhatsApp number, receive a 6-digit OTP via WhatsApp message from the Omeru bot, then enter it to authenticate. JWT is issued on success. Full-screen split layout: dark left panel (Omeru branding, feature callouts) + off-white right panel (form).

Left Panel (dark, --dark-gray)

  • Omeru wordmark in lime
  • Display headline: "Your store. Your control."
  • 3 benefit bullets with lime check marks
  • Noise texture overlay
  • Floating WA bubble mockup (animated)

Right Panel (--off-white)

  • "+27" country prefix + phone input
  • "Send OTP via WhatsApp" btn-lime
  • OTP 6-digit input (auto-focus each digit)
  • Resend countdown (30s)
  • Terms link (small, mid-gray)
Dashboard Home
/dashboard
Port + Extend

The merchant's home screen. Loads in under 1s with today's KPIs, a 7-day revenue sparkline, the last 5 orders, and top products. The shop open/close toggle lives permanently in the top-right — the single most important action a merchant takes daily.

Top Section

  • Store name (Syne 700, 24px)
  • Open/Close toggle (lime dot + label)
  • Today's date + notification bell
  • Pending orders badge (lime if > 0)

KPI Strip (4 cards)

  • Today Revenue (Syne 800, lime-on-dark)
  • Today Orders
  • Pending Orders (featured/highlighted)
  • Upcoming Bookings

Revenue Chart

  • Area/bar chart, 7 days
  • Lime bars, off-white bg card
  • Hover tooltip with day + amount
  • 7d vs 30d tab toggle

Bottom Row (2 tables)

  • Recent Orders: ID, total, status pill, action
  • Top Products: name, units sold this month
  • Quick actions: Add Product, View Kitchen
Orders — Kitchen
/orders
Port + Extend

The operational core. Merchants manage the full order lifecycle here. Default view is a 4-column Kanban board. A list/table toggle is available. Clicking any order opens a right-side detail panel. Status transitions send WhatsApp notifications to the customer automatically — same as the bot.

Toolbar

  • Status tab filter (All / Pending / Paid / Ready / Done)
  • Date range filter
  • List vs Kanban toggle
  • Export CSV button

Kanban Board (4 columns)

  • Pending — awaiting payment
  • To Prepare — paid, needs fulfillment
  • Ready for Pickup — awaiting collection
  • Done / Cancelled — archive

Order Card

  • Order ID (last 5 chars, monospace)
  • Total (Syne, bold)
  • Item count + time elapsed badge
  • One-click status advance button

Detail Panel (slide-in right)

  • Full item list with quantities and prices
  • Order total
  • Status timeline (OrderStatusHistory)
  • Mark Ready / Complete / Cancel buttons
  • Cancellation reason input field

Add / Edit Product
/products/new  ·  /products/[id]
Port + Extend

Full-page form replacing the 4-step WhatsApp conversation. Two-column layout: left column has the image dropzone and preview; right column has all text fields. Variants are managed in a simple table beneath the main fields. No pagination — everything on one scrollable page.

Left — Image

  • Large drag-and-drop dropzone
  • Click to open file picker
  • Image preview on upload
  • Remove image link

Right — Fields

  • Name (text, 2–50 chars)
  • Price (number, currency prefix)
  • Category (select + create new inline)
  • Description (textarea, 500 char limit)
  • In-stock toggle

Variants Section

  • Expandable table (hidden by default)
  • Columns: Size, Color, SKU, Price, In-stock, Delete
  • "+ Add Variant" row
  • Inline editing in cells

Action Bar (sticky bottom)

  • "Save Draft" btn-outline
  • "Publish" btn-lime
  • "Archive" (if editing existing)
  • Back link (top left)

/services
Port + Extend

Service catalog for merchants with services_enabled = true. Shows all services as cards with active/hidden toggle. Pending booking requests appear as an alert strip at the top. Non-enabled merchants see a locked state with an "Apply to offer services" CTA.

Service Card

  • Service image (or placeholder icon)
  • Name + duration badge (e.g. "60 min")
  • Price
  • Upcoming confirmed bookings count
  • Visible / Hidden toggle
  • Edit / Delete actions

Pending Requests Alert

  • Lime banner: "3 booking requests need your response"
  • Quick-view inline or link to /bookings

Add Service Flow (modal)

  • Step 1: Name
  • Step 2: Price
  • Step 3: Duration (preset buttons)
  • Step 4: Image upload
  • Publish to activate

Locked State

  • Blurred card placeholders
  • "Services not enabled" overlay
  • "Apply to offer services" btn-lime
  • Application form: category + description

Calendar Header

Time Grid

Right Panel — Requests

  • Pending requests list (oldest first)
  • Customer name, service, date/time
  • Note if provided
  • Accept btn-lime / Decline btn-outline

Decline Modal

  • Quick pick: "Fully booked" / "Not available"

Analytics
/analytics
New — Untapped Data

The most data-rich page in the portal. Surfaces PlatformEvent and SearchQuery data for the first time — merchants have never seen storefront view counts, WA tap rates, or what customers search for. Date range picker (7d / 30d / 90d / all-time) applies globally to all charts.

Summary KPIs (top strip)

  • Total Revenue (selected period)
  • Total Orders
  • Avg Order Value
  • Total Customers

Revenue Chart (full width)

  • Area chart, daily data points
  • Lime fill, dark-gray axis labels
  • Period tab: 7d / 30d / 90d / All
  • Hover tooltip: date + revenue

Top Products (bar chart)

  • Horizontal bars, sorted by units sold
  • Product name + image thumbnail
  • Units sold + revenue per product

Storefront Events (PlatformEvent)

  • Product views per item
  • WA click-through rate
  • Cart add events
  • First time ever surfaced to merchants

Search Queries (SearchQuery table)

  • What customers type into storefront search
  • Query, count, results returned
  • Zero-result queries (inventory gaps)

Order Status Breakdown

  • Donut chart: completed vs cancelled
  • Completion rate % highlight
Customers
/customers
New

Full customer roster derived from MerchantCustomer. Merchants have never had visibility into individual customer data — this is the first time they can see who their regulars are, total spend, and last interaction. Clicking a row opens a detail panel with order history.

Toolbar

  • Search by name or number
  • Filter: All / Bookmarked / Opted Out
  • Sort: Spend ↓, Last Seen ↓, Orders ↓

Customer Table

  • WA number (masked: +27 *** *** 4567)
  • Display name (if set)
  • Province
  • Order count
  • Total spend (formatted currency)
  • Last seen (relative time)
  • Bookmark star icon

Detail Panel

  • Full WA number
  • Order history list (last 10)
  • Booking history (if applicable)
  • Opt-out status indicator
  • Bookmark toggle
Broadcast
/broadcast
Port + Extend

Compose and send a WhatsApp message to all opted-in customers. Replaces the raw text-in-WhatsApp approach with a proper rich-text composer, live recipient count, message preview with opt-out footer, and confirmation step before send.

Composer

  • Large textarea (400px min-height)
  • Character count (1024 max)
  • Opt-out footer preview (auto-appended)
  • Recipient count: "Sending to X customers"

Preview Panel (right)

  • WA bubble mockup (wa-bubble class)
  • Live preview as you type
  • Opt-out footer shown in muted text

Confirm & Send

  • "Send to X customers" btn-lime
  • Confirmation modal with final count
  • Progress indicator during send
  • Success: "Sent to X / Y" result

History Section (below)

  • Past broadcasts from AuditLog
  • Date, message preview, sent count
  • Pagination (5 per page)
Settings
/settings
Port + Extend

Tab-based settings page covering all merchant configuration currently spread across 6+ WhatsApp menus. Four tabs: Profile, Hours, Team, and Store. Each tab is a clean form — no conversation-style flow, just fields, toggles, and file pickers.

Tab: Profile

  • Trading name
  • Description / bio (textarea)
  • Logo upload (dropzone)
  • Welcome image upload
  • Address (text + optional map pin)
  • Support number (E.164 input)
  • Welcome message (textarea)

Tab: Hours

  • Mon–Fri: open/close time pickers
  • Saturday: open/close time pickers
  • Sunday: toggle only
  • "Restore standard hours" button
  • Live preview of hours schedule

Tab: Team

  • Current owners table (WA, role, joined date)
  • Pending invites list
  • "Invite by WA number" btn-lime
  • Remove owner (OWNER role only)
  • Role badge: OWNER / ADMIN / STAFF

Tab: Store

  • Handle + admin handle (edit with confirmation)
  • Store category picker (grid of icons)
  • Currency dropdown (ISO codes)
  • Locale dropdown (en-ZA, etc.)
  • Browse visibility toggle
  • Brand name field
Reviews
/reviews
Port

Customer feedback collected after every completed order (via WhatsApp rating prompt). Read from AuditLog where action = CUSTOMER_FEEDBACK. The WhatsApp bot shows paginated text; the web portal shows a proper review feed with star distribution and average rating.

Rating Overview (top)

  • Average stars (Syne 800, large)
  • Star distribution bars (5★ → 1★)
  • Total review count

Review Feed

  • Star rating (⭐ icons)
  • Comment text (if provided)
  • Date (relative + absolute)
  • Paginated, newest first
Onboarding / KYC Status
/onboarding
New

Shown to merchants with status = ONBOARDING. A guided checklist of steps to get the store live: add products, complete KYC, set trading hours. Pulls from the kyc_online_completed and kyc_submitted_at fields. Disappears once status = ACTIVE.

Progress Strip

  • Horizontal stepper (3–5 steps)
  • Completed: lime checkmark
  • Current: lime border, pulsing dot
  • Remaining: mid-gray

Steps

  • 1. Add your first product
  • 2. Set your trading hours
  • 3. Complete KYC verification
  • 4. Go live

Recommended Tech Stack: Next.js 15 App Router · TypeScript · Tailwind (same globals.css tokens) · Prisma (shared schema — same DB) · Recharts for analytics · next-auth or custom JWT for OTP session · SSE for real-time order stream.
